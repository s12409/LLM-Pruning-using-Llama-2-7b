# llm_final_project

Environments:

ubuntu 22.04

Python 3.10.15

torch = 2.5.1+cu121

# Installation

```jsx
# create conda env
conda create --name llm_project python==3.10
conda activate llm_project

# install pytorch
pip3 install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118

# install LLM_pruner
cd LLM-Pruner
pip install -r requirement.txt
cd ..

# install qlora
git clone https://github.com/artidoro/qlora.git
cd qlora
pip install -U -r requirements.txt
cd ..

# install lm-evaluation-harness=0.3.0
cd lm-evaluation-harness
pip install -e .
cd ..
```

# LLAMA2-7B inference

model is in  `llama-2/7B`

you can use `python model_inference.py` to check model inference

and use `def __test_model_logits` function to change model:

### errors:

`AttributeError: undefined symbol: cquantize_blockwise_fp16_nf4  
pip install -U bitsandbytes`

# LLM Pruner

https://github.com/horseee/LLM-Pruner?tab=readme-ov-file#limitations

you can use `python prune_llm_pruner.py` to prune the model and set parameters at main functions

Note: cuda out of memory  when using RTX3060 12GB 

So we prured the model by CPU (i5-12400) and 64 GB RAM (training time = 8h 30 min)

Set 2 different ratio 0.25, 0.5

To optimize computational efficiency for the hardware, the model shapes are adjusted to multiples of 64.

use `python prune_model_expand_to64.py`  to get expand model

# Model evaluate

## PPL

`python prune_llm_pruner.py` to evaluate ppl

using `wikitext` datasets

# lm_eval

`python eval_lm_eval.py` to evaluate different tasks
`task` to change different task
`test_model` to change different model

# Rouge test

use hw2 dataset

`python eval_hw2_rouge.py` to evaluate rouge same as hw2

you may need to change **batch size** to avoid CUDA OOM

notice that results generated by llama2-7B are worse than hw2 results

# qlora

To fine-tuning model by hw2 datasets

tran.csv needs to be convert as {’input’,’output’} format 

thus use `python datasets_finetune.py` to convert

to run qlora 

```jsx
cd qlora
python qlora.py --model_name_or_path ../llama-2/7B --dataset="../hw2_datasets/train.json"
```

notice: when using RTX3060 12GB 

Needs to train about 180 hours to achieve `optimizer update steps = 10000`

We get checkpoints every 250 steps

To evaluate: 

```jsx
lora_adapter = model.load_adapter(lora_model_path)
model.active_adapters = lora_adapter
```

load check points and set as model adapter
